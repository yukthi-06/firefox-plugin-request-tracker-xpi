# Walkthrough: Addition of "Stop Capture" / "Start Capture" Button

We have added a new toggle control in the extension's popup logger interface to allow users to pause or resume network traffic capture directly from the UI.

## Changes Made
- **UI Element**: Added a `<button id="toggle-capture-btn">` in [logger.html](file:///x:/Projects_X/0_Active/A_XPI_Firefox_Plugin/firefox-plugin-request-tracker-xpi_GITHUB/extension/logger.html) adjacent to the "Clear Session" button.
- **Dynamic Styling**: Added `.pulse-indicator.inactive` rules in [logger.css](file:///x:/Projects_X/0_Active/A_XPI_Firefox_Plugin/firefox-plugin-request-tracker-xpi_GITHUB/extension/logger.css) to turn the pulsing green dot grey and stop the animation when capturing is disabled.
- **Control Logic**: Updated [logger.js](file:///x:/Projects_X/0_Active/A_XPI_Firefox_Plugin/firefox-plugin-request-tracker-xpi_GITHUB/extension/logger.js) to bind click events to the button, send a `{ action: "toggleTracking" }` message to the background script, and update the button styling and label ("Stop Capture" vs "Start Capture") dynamically based on the current state.
- **Message Exchange**: Updated the background script in [background.js](file:///x:/Projects_X/0_Active/A_XPI_Firefox_Plugin/firefox-plugin-request-tracker-xpi_GITHUB/extension/background.js) to accept the toggle message, invert the `trackingActive` state, and return the new status back to the popup.
