# User Prompt
```
Create a complete production-quality Mozilla Firefox WebExtension (Manifest V3 if supported by Firefox, otherwise use the latest Firefox-compatible extension architecture) named "Session Resource Logger".

## Goal

The extension (when activated) continuously records:

1. Every webpage URL visited by the user.
2. Timestamp of each visit.
3. Every network resource loaded by those pages, including:

   * Images
   * CSS
   * JavaScript
   * Fonts
   * Audio files
   * Video files
   * XHR/Fetch requests
   * WebSocket connections (if available)
   * Other external resources

The extension must maintain a complete in-memory session log during the browser session and display it in a dedicated popup window.

---

## Functional Requirements

### Page Visit Tracking

Whenever a page is loaded or navigated:

Capture:

* Full page URL
* Page title (if available)
* Timestamp
* Tab ID
* Window ID

Store an entry similar to:

{
"type": "page",
"timestamp": "2026-06-09T10:15:30.123Z",
"url": "https://example.com/products",
"title": "Products",
"tabId": 4,
"windowId": 1
}

The logging must continue for the entire browser session.

---

### Resource Tracking

For every resource requested by visited pages:

Capture:

* Parent page URL
* Resource URL (full URL)
* Resource type
* Timestamp
* HTTP response status code
* HTTP method
* Content-Length (if available)
* File size in bytes
* MIME type
* Domain
* Request ID
* Tab ID

Examples:

https://cdn.example.com/style.css
https://cdn.example.com/app.js
https://example.com/image.jpg
https://media.example.com/audio.mp3

Store entries such as:

{
"type": "resource",
"timestamp": "2026-06-09T10:15:31.100Z",
"pageUrl": "https://example.com/products",
"resourceUrl": "https://cdn.example.com/app.js",
"resourceType": "script",
"statusCode": 200,
"method": "GET",
"mimeType": "application/javascript",
"fileSize": 154321,
"domain": "cdn.example.com",
"tabId": 4
}

Use Firefox WebRequest APIs to capture request and response information.

If Content-Length is unavailable, store:

fileSize = -1

Do not fail.

---

### Dedicated Logging Window

The extension must create a separate popup window (not the browser action popup).

Behavior:

* Automatically open when the extension starts.
* Remain open throughout the browsing session.
* If closed by the user:

  * Automatically reopen on next logged event.
  * Or provide a reopen button in the extension toolbar.

Window title:

Session Resource Logger

Dimensions:

width: 1400
height: 900

Use browser.windows.create().
```
