# Walkthrough: Window Closure Behavior Fix

We have updated the extension's window lifecycle management in [background.js](file:///x:/Projects_X/0_Active/A_XPI_Firefox_Plugin/firefox-plugin-request-tracker-xpi_GITHUB/extension/background.js) to resolve the issue where closing the window caused it to reappear immediately on the next logged event.

## Changes Made
- **Deactivate Tracking on Close**: When the user closes the **Session Resource Logger** popup window, the `onRemoved` listener triggers and automatically sets `trackingActive = false` (saving this to storage). This immediately halts all background navigation and web request logging.
- **Removed Auto-Reopen**: Removed the automatic reopening routine inside the `addLogEntry` function. The window will now remain closed, and tracking will remain suspended.
- **Reopen / Restart Tracking**: The user can resume tracking and open the window at any time by clicking the extension action icon in the top toolbar of Firefox.
